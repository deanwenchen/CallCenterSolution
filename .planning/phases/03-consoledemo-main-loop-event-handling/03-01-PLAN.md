Add two new ProcessResult subtypes and their factory methods to the existing ProcessResult hierarchy in EntryPoint.cs (lines 15-24):

1. Add record types:
   - `public record TimeoutResult(bool IsWarning, string Message) : ProcessResult;`
     - IsWarning = true → 30 min timeout, show warning but allow continuation
     - IsWarning = false → 60 min timeout, force terminate
   - `public record IntentSwitchResult(string OldWorkflow, string NewIntent) : ProcessResult;`

2. Add factory methods to ProcessResult:
   - `public static ProcessResult TimeoutWarning(string msg) => new TimeoutResult(true, msg);`
   - `public static ProcessResult TimeoutTerminate(string msg) => new TimeoutResult(false, msg);`
   - `public static ProcessResult IntentSwitch(string oldWf, string newIntent) => new IntentSwitchResult(oldWf, newIntent);`

3. Add CheckTimeoutAsync method to EntryPoint class:
   ```
   public async Task<ProcessResult?> CheckTimeoutAsync(string sessionId, CancellationToken ct = default)
   {
       var lastActivity = await GetLastActivityAsync(sessionId, ct);
       if (lastActivity == null) return null; // First input, no timeout check

       var elapsed = DateTime.UtcNow - lastActivity.Value;

       if (elapsed.TotalMinutes >= 60)
       {
           await ClearActiveWorkflowAsync(sessionId, ct);
           return ProcessResult.TimeoutTerminate("Session terminated due to 60 minutes of inactivity.");
       }

       if (elapsed.TotalMinutes >= 30)
       {
           await ClearActiveWorkflowAsync(sessionId, ct);
           return ProcessResult.TimeoutWarning("Warning: 30 minutes of inactivity. Active workflow has been cleared. You can start a new conversation.");
       }

       return null;
   }
   ```

   Per D-05 and D-06: This is passive — only called when user provides input. No background timer. Both 30-min warn and 60-min terminate clear activeWorkflow.

4. Modify ProcessAsync to detect intent switch (D-03, D-04):
   After the existing activeWorkflow check (line 91-98), before returning ResumeExisting, compare the recognized intent with the activeWorkflow:

   - If activeWorkflow == "RefundWorkflow" AND intent.Intent != "refund" (and not greeting/unknown):
     → This is an intent switch. Clear activeWorkflow, return IntentSwitchResult("RefundWorkflow", intent.Intent).
   - If activeWorkflow == "RefundWorkflow" AND intent.Intent == "refund":
     → Same intent, proceed with ResumeExisting as before.
   - Per D-04: Phase 3 only does refund, so any non-refund intent during active RefundWorkflow = switch.

   The modified ProcessAsync flow:
   a. Set lastActivity timestamp (from Task 1)
   b. Check timeout first — if timeout result, return it immediately
   c. Check if activeWorkflow exists:
      - If yes, recognize intent
      - If intent == "refund" → ResumeExisting (same workflow)
      - If intent != "refund" and != "greeting"/"unknown" → IntentSwitch (clear activeWorkflow, return switch result)
      - If intent == "greeting"/"unknown" → NoIntent (don't clear activeWorkflow, let user chat)
   d. If no activeWorkflow, recognize intent and start new workflow or reply naturally (existing logic, unchanged)

   Keep existing RecognizeIntentAsync, GetActiveWorkflowAsync, SetActiveWorkflowAsync, ClearActiveWorkflowAsync methods unchanged.

   Per IR-03: greeting/unknown intents during active workflow should NOT trigger intent switch — they should return NoIntentResult with a natural reply, leaving the active workflow intact. The user can continue chatting and then resume the workflow later.
