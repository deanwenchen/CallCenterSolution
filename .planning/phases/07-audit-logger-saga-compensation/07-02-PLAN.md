### Task 2: Wire Saga compensation into ConsoleDemo RunWorkflow

<read_first>
- src/CallCenter.ConsoleDemo/Program.cs (WorkflowErrorEvent case, lines 135-144)
- src/CallCenter.Framework/Saga/SagaBuilder.cs (from Plan 01)
- src/CallCenter.Workflows/Refund/Executors/RestoreCouponExecutor.cs (compensation executor)
</read_first>

<action>
In `Program.cs` RunWorkflow's `WorkflowErrorEvent` case:

Current behavior (lines 135-144): print error, clear activeWorkflow, return

New behavior with Saga:
1. Check if error source is "ExecuteRefund" (executor ID)
2. If yes, create SagaBuilder:
   ```
   var saga = new SagaBuilder()
       .OnFailure("ExecuteRefund", async ct => await restoreCoupon.HandleAsync(...))
       .WithRetry(3, TimeSpan.FromSeconds(1), TimeSpan.FromSeconds(2), TimeSpan.FromSeconds(3));
   await saga.ExecuteAsync(async ct => { /* retry original refund */ });
   ```
3. If compensation succeeds → print "补偿完成"
4. If compensation fails → print "补偿失败: {exception.Message}"
5. Audit both the original failure and the compensation via AuditLogger

The restoreCoupon executor needs to be accessible from RunWorkflow — add it as a parameter or capture from outer scope.
</action>

<acceptance_criteria>
- WorkflowErrorEvent from ExecuteRefund triggers RestoreCoupon compensation
- Retry delays use 1s/2s/3s (demo acceleration)
- Compensation success/failure printed to console
- Audit entries for both failure and compensation
- Build: 0 errors
</acceptance_criteria>
