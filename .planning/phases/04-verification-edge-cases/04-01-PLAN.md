修改 ExecuteRefundExecutor.cs 的 `!message.Confirmed` 分支。

当前代码（第 23-27 行）：
```csharp
if (!message.Confirmed)
{
    await context.SendMessageAsync(RefundSignal.Cancelled, cancellationToken: cancellationToken);
    throw new InvalidOperationException("User cancelled refund");
}
```

替换为：不调用财务服务、不发送 RefundSignal.Cancelled、不抛异常。改为：
1. 通过 `context.YieldOutputAsync(new RefundNotification("退款已取消"), cancellationToken)` 输出取消通知
2. 返回一个表示取消状态的 `RefundExecuted`（或等价返回类型），让工作流正常结束

关键决策（per D-69/D-70）：
- 不发送 RefundSignal.Cancelled — 用户说取消时工作流已经过了 ConfirmPort
- YieldOutputAsync 用于终端输出（已有模式：SendNotificationExecutor 和 RefundDeniedExecutor 都用此模式）
- 不抛异常 — 流程正常终止，不是错误状态

注意：RefundExecuted 当前定义为 `new RefundExecuted(result)` 需要 RefundResult。如果取消路径没有 RefundResult，考虑：
- 选项 A：让 RefundExecuted 支持可选 result 或添加 isCancelled 属性
- 选项 B：返回一个特殊标记的 RefundExecuted（如 null result）
- 根据 MAF 工作流图定义选择最符合图结构的方式

参考 SendNotificationExecutor.cs 和 RefundDeniedExecutor.cs 中 YieldOutputAsync 的使用模式。
