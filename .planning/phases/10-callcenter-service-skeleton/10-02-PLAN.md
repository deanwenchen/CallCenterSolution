在 src/CallCenter.AgentHost/CallCenterService.Intent.cs 中创建 `public partial class CallCenterService` 的 `ProcessAsync(string sessionId, string userMessage, CancellationToken ct = default)` → `Task<string>` 方法。

**文件头部**包含 `#pragma warning disable MAAI001`（与 Program.cs / EntryPoint.cs / Core.cs 保持一致，因为使用 Microsoft.Agents.AI 实验性 API）。

此方法整合 ProcessResult 的 dispatch 逻辑（从 Program.cs 第 123-154 行的 switch 迁移而来）：

1. 调用 `await ResolveWorkflow(sessionId, userMessage, ct)` 获取 ProcessResult
2. switch result:
   - **ResumeExistingResult** → 调用 `await ResumeWorkflowAsync(sessionId, userMessage, ct)`（Execution.cs 方法，Wave 3 实现，当前可用 `throw new NotImplementedException("ResumeWorkflowAsync not yet implemented")` 占位，但注释说明 Wave 3 会实现）
   - **TimeoutResult timeout** → 返回 timeout.Message（如果 IsWarning 则返回带警告前缀的消息）
   - **IntentSwitchResult switchResult** → 返回 $"已终止 {switchResult.OldWorkflow} 流程，新意图 '{switchResult.NewIntent}' 暂未实现"
   - **NoIntentResult noIntent** → 返回 noIntent.Response
   - **StartWorkflowResult start** → 调用 `await RunWorkflowAsync(sessionId, start.InitialMessage, ct)`（Execution.cs 方法，Wave 3 实现，同样可用 NotImplementedException 占位）
3. 返回最终的 string 结果

注意：ProcessAsync 的签名必须是 `public async Task<string> ProcessAsync(string sessionId, string userMessage, CancellationToken ct = default)`（per CS-01 规格：返回 string，内部完成意图识别→工作流执行→返回结果）。

注意：调用 `RunWorkflowAsync` 和 `ResumeWorkflowAsync` 时，`sessionId` 必须作为第一个参数传递（与 Execution.cs 的方法签名一致）。当前 Wave 2 阶段，这些方法尚未实现（Wave 3），所以调用它们时返回占位字符串即可。但 ProcessAsync 的路由 dispatch 逻辑必须完整，且占位调用中必须包含 `sessionId` 参数。
