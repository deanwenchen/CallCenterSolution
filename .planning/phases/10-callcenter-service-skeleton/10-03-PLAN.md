**方法 1: `RunWorkflowAsync(string sessionId, RefundIntent initialMessage, CancellationToken ct)` → `Task<string>`**
替代 Program.cs 的 RunWorkflow 方法（line 159-286）。`sessionId` 用于 session store 读写、审计日志、EventBus 调用。

1. 初始化 `CheckpointInfo? lastCheckpoint = null`
2. 初始化 `var currentMessage = initialMessage`
3. while(true) 循环：
   a. `await using var run = await InProcessExecution.RunStreamingAsync(_refundWorkflow, currentMessage, _checkpointManager)`
   b. `bool needsOrderId = false`
   c. `await foreach (WorkflowEvent evt in run.WatchStreamAsync())` — 调用 `HandleEventAsync(sessionId, evt, run, ref lastCheckpoint, ref needsOrderId, ref currentMessage, isResumeMode: false, ct)`，如果返回 EventResult.Return（表示工作流已完成），则 break 内层循环并返回结果

   e. 如果 needsOrderId，继续 while 循环重跑（与 Program.cs line 284 逻辑相同）

**方法 2: `ResumeWorkflowAsync(string sessionId, string userMessage, CancellationToken ct)` → `Task<string>`**

1. 从 session store 读取断点：`var checkpoint = await _sessionStore.GetAsync<CheckpointInfo>("lastCheckpoint", sessionId, ct)`
2. 如果 checkpoint 为 null，返回 "未找到断点记录，请重新启动流程"
3. `await using var run = await InProcessExecution.ResumeStreamingAsync(_refundWorkflow, checkpoint, _checkpointManager)`
4. `CheckpointInfo? lastCheckpoint = checkpoint`
5. `await foreach (WorkflowEvent evt in run.WatchStreamAsync())` — 调用 `HandleEventAsync(sessionId, evt, run, ref lastCheckpoint, ref unusedNeedsOrderId, ref unusedCurrentMessage, isResumeMode: true, ct)`

**方法 3: `DriveLoopAsync` — 共享事件循环**（per design.md Decision 6）

由于 RunWorkflow 和 ResumeWorkflow 的初始化不同（RunStreamingAsync vs ResumeStreamingAsync）但事件处理逻辑 ~50% 重复，抽取共享的 HandleEventAsync 方法。DriveLoopAsync 的职责是：
- 接受 `IStreamingRun run` 参数（RunWorkflow 和 ResumeWorkflow 都创建这个）
- 遍历 `run.WatchStreamAsync()` 的每个 WorkflowEvent
- 对每个事件调用 HandleEventAsync
- 返回 EventResult（Continue / Return / Retry）

签名：`private async Task<EventResult> DriveLoopAsync(string sessionId, IStreamingRun run, bool isResumeMode, ref CheckpointInfo? lastCheckpoint, ref bool needsOrderId, ref RefundIntent? currentMessage, CancellationToken ct)`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Console.In → HandleRequestAsync | 用户输入直接进入服务层，无输入验证（控制台场景可接受） |
| WorkflowEvent → HandleEventAsync | 工作流事件来自 Microsoft.Agents.AI.Workflows 框架，信任框架输出 |
| ExternalRequest → HandleRequestAsync | 工作流发出的请求，信任请求结构但需验证端口类型 |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Repudiation | AuditLogger logging | mitigate | 所有 9 种事件类型都通过 AuditTrailMiddleware 或 _auditLogger.LogAsync 记录，确保操作可追溯。WorkflowOutputEvent 结束时调用 VerifyChainAsync 验证审计链完整性 |
